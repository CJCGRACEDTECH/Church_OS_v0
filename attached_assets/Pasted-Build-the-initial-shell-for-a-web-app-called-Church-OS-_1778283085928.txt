Build the initial shell for a web app called “Church OS.”

This is NOT the full product yet. The goal is to create a stable starting point with simple login functionality for two user types: Admin and Member.

Focus on stability, clean structure, and testable login behavior. Do not overbuild features. Most app features should be visible but disabled/greyed out as placeholders.

Project Goal:
Create a working Church OS app shell where I can:
1. Open the app
2. See a landing/login page
3. Login as an Admin using seeded test credentials
4. Login as a Member using seeded test credentials
5. Be redirected to the correct dashboard based on role
6. See placeholder navigation items for future modules
7. Logout successfully
8. Verify the app runs without errors

Tech Stack:
- React frontend
- Node.js + Express backend
- PostgreSQL database
- Tailwind CSS
- Use simple JWT-based auth or session-based auth
- Keep the implementation simple and stable

Important:
- Do not integrate Stripe yet

Security Requirements:
- Passwords must be hashed
- Do not store plain text passwords
- Use environment variables for secrets
- Add .env.example with required variables
- Do not expose backend secrets to the frontend
- Basic route protection should work

Backend Requirements:
Create a simple Express backend with the following routes:

1. POST /api/auth/login
Purpose:
- Accept email and password
- Validate user
- Return token/session and user role

2. GET /api/auth/me
Purpose:
- Return current logged-in user if token/session is valid

3. POST /api/auth/logout
Purpose:
- Clear session/logout behavior

4. GET /api/health
Purpose:
- Return basic status that backend is running
